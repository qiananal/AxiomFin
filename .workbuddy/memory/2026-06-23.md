# 2026-06-23 工作日志

## 修复 AxiomFin 股价监控 Bug

### Bug 1: send_email_report 邮件内容被测试文本覆盖
- **根因**: `core_tools.py` 第58-65行硬编码了 `test_subject` 和 `test_content`，导致 `subject` 和 `report_content` 参数被丢弃
- **修复**: 删除测试文本，直接使用传入的 `subject`（Header）和 `report_content`（HTML格式）
- **文件**: `tools/core_tools.py`

### Bug 2: cron_scheduler_loop 中 price=0 导致无限重复查询
- **根因**: 当 TDX/新浪 API 返回 price=0 时，`if current_price > 0:` 跳过所有触发逻辑，任务保持 ACTIVE，每10秒重试 → 死循环
- **修复**:
  - 增加连续失败计数器 `fail_counter`，连续失败 5 次自动标记为 `ERROR_TRIGGERED`
  - 分别处理 API 返回 error 字段和 price=0 两种情况
  - 成功获取价格后重置失败计数器
  - 轮询间隔从 10s 提升至 60s，减少通达信节点压力
- **文件**: `app_frontend.py`

### Bug 3: 文件竞态条件
- **根因**: `add_monitor_task` 和 `cron_scheduler_loop` 各自独立读写 `monitor_tasks.json`，无共享锁
- **修复**: 在 `core_tools.py` 中新增 `MONITOR_FILE_LOCK = threading.Lock()`，两端共用同一把锁
- **文件**: `tools/core_tools.py`, `app_frontend.py`

### 附加改进
- 删除 `send_email_report` 函数内冗余的 `import smtplib/MIMEText/Header`
- 邮件格式改为 HTML，增加表格化报警内容
- Sidebar 状态增加 `ERROR_TRIGGERED` 显示
- 巡检日志更清晰：区分"API查询失败""价格为空""未触发""已触发"四种状态

### Bug 4: QQ邮箱 SMTP 550 From header 被拒
- **根因**: `Header(f"AxiomFin 智能监控 <{email}>", "utf-8")` 把整串（含邮箱地址和尖括号）全部 Base64 编码，违反 RFC 5322，QQ 服务器直接 550 拒绝
- **修复**: 用 `email.utils.formataddr(("AxiomFin 智能监控", sender_email))` 替代，仅编码显示名部分，邮箱地址保持 ASCII 原文
- **文件**: `tools/core_tools.py` (新增 import + 修改第59-61行)
